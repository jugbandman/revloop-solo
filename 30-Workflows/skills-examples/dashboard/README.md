# Dashboard — React Pipeline Visualization

Ship a React dashboard that reads your vault's markdown frontmatter and renders pipeline charts, enrichment scores, and prospect lists. No backend required. The app runs locally, reads vault files directly from the filesystem, and exports a static JSON snapshot for sharing.

## The Pattern

**Stack:** Vite + React + Tailwind CSS + Recharts + Zustand

The app does not have a server. It reads markdown files at build time (or on demand via a watcher script), extracts frontmatter as JSON, and stores that JSON in a file the React app imports. The UI then renders charts and tables from that data.

**Data flow:**
1. A Python script (`export-dashboard-data.py`) scans `80-Pipeline/` and writes `dashboard/src/data/pipeline.json`
2. Vite bundles the JSON into the app at build time
3. The React app reads from Zustand store, which hydrates from `pipeline.json`
4. Recharts renders the charts; Tailwind handles layout

For live updates during development, run `export-dashboard-data.py --watch` in a terminal alongside `vite dev`. The script uses `watchdog` to detect file changes and re-export.

## Key Components

**Pipeline funnel chart**
A Recharts `BarChart` or `FunnelChart` showing deal count by stage (lead, prospect, qualified, proposal, closed). Data comes from `stage:` frontmatter across `80-Pipeline/deals/` and `80-Pipeline/prospects/`.

**Enrichment score table**
A sortable table of leads with `clay-score:` frontmatter populated. Columns: name, company, score, tier, signals, last-touched. Filter by tier (A/B/C) or score range.

**Outreach status board**
A card grid or table showing HeyReach campaign status per prospect. Data from `heyreach-status:` and `heyreach-campaign:` frontmatter. Grouped by campaign name.

**Activity timeline**
A Recharts `LineChart` or `AreaChart` showing deals touched per week, derived from `last-touched:` dates across all pipeline files.

## Getting Started

This README documents the pattern only. No code is included here because the component structure and data shape are too project-specific to port generically. When you're ready to build:

1. Run `npm create vite@latest dashboard -- --template react-ts` from the vault root
2. Install deps: `npm i recharts zustand tailwindcss @headlessui/react`
3. Copy `export-dashboard-data.py` from `30-Workflows/skills-examples/enrichment/clay-enrich.py.example` as a reference for the frontmatter parsing pattern
4. Build `export-dashboard-data.py` to scan `80-Pipeline/` and write JSON
5. Build the Zustand store to load `pipeline.json`
6. Build charts one at a time, starting with the funnel (simplest data shape)

The enrichment scripts in `30-Workflows/skills-examples/enrichment/` include a reference implementation of frontmatter parsing that you can reuse in `export-dashboard-data.py`.

## Sharing

To share a snapshot with someone who doesn't have the vault:
1. Run `export-dashboard-data.py` to refresh the JSON
2. Run `vite build` to produce a static site in `dashboard/dist/`
3. Zip and send, or deploy to Vercel/Netlify with `vercel --prod dist/`
